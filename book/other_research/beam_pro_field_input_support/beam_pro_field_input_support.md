# Beam Proアプリの試作

## はじめに

本プロジェクトは、XREAL と Beam Pro を活用し、現場作業における入力負荷を下げることを目的に試作しているアプリケーションである。紙やスマートフォン中心の運用では、視線移動や持ち替え、記録漏れ、確認の遅れが発生しやすい。そこで、視界に近い位置で情報を扱えるデバイスと、現場で扱いやすい入力フローを組み合わせることで、より自然な作業支援の形を探っている。

> - 現在はプロトタイプ段階であり、今後は実運用に向けた改善を進める


## 開発の背景

現場での記録や確認は単純なようでいて、実は多くの手間を伴う。作業対象を見ながら記録を残したい場面でも、従来は紙にメモを取る、スマートフォンを開く、別端末で確認するといった行き来が発生しがちである。こうした小さな中断が積み重なることで、作業効率や入力品質に影響が出る。

本プロジェクトでは、ウェアラブル表示とモバイル実行環境を組み合わせたとき、現場入力の体験がどこまで改善できるかを検証している。狙いは、作業者の負担を増やさずに、入力・確認・記録・支援情報の受け取りを一つの流れとして成立させることである。

## 何を作っているのか

開発しているのは、Beam Pro 上で動作する Unity ベースの作業入力支援アプリである。作業者は現場の状況を端末で取り込み、短いメモや操作入力を行い、その内容に応じた判断補助情報を同じ画面の中で確認できる。さらに、必要に応じて結果を音声でも受け取れるようにしている。

入力した内容や取り込んだデータはセッション単位で保存され、後から振り返ることも前提にしている。すなわち、現場での一時的な支援だけでなく、作業記録の整備という観点も含んだ試作である。

## システム構成

全体構成はできるだけシンプルにしている。現場側では Beam Pro 上のアプリが入力と表示を担い、必要に応じて端末のカメラや音声機能を利用する。取得した情報はローカルに保存されるとともに、クラウド側のワークフローに連携され、判断補助のための結果が返される構成である。

## 現状できること

現時点の試作で実現できていることは次のとおりである。

- Beam Pro 上で、作業入力向けの専用画面を表示できる
- 端末カメラを使って現場の情報を取り込み、操作フローに組み込める
- 入力内容をセッション単位で保存し、後から参照できる
- クラウド側の処理基盤と連携し、判断補助のための結果を取得できる
- 取得した結果を画面表示でき、音声読み上げにも対応できる

この段階でも現場における「入力して終わり」ではなく、「入力して、その場で確認し、必要な支援を受ける」という一連の流れは見え始めている。特に画面遷移を増やさず同一画面内で状態が進んでいく設計は、本プロジェクトの特徴である。

## 今後の課題

一方で、現時点ではまだ試作段階であり、そのまま現場導入できる完成度には達していない。今後の課題は以下のとおりである。

- 実運用を前提とした UI の磨き込み
- 通信状況や権限処理を含む安定性の向上
- 現場ごとの業務フローに合わせた入力項目の最適化
- 保存データの取り扱いに関するセキュリティと運用設計
- 実際の利用者による評価を踏まえた改善サイクルの確立

特に重要なのは技術的に動くことと、現場で使い続けられることは別であるという点である。表示の分かりやすさ、入力の少なさ、確認のしやすさ、応答速度といった要素はすべて実用性に直結する。今後は機能追加そのものよりも、現場適合性を高める改善が中心になる。

## おわりに

本プロジェクトは、XREAL と Beam Pro を活用し、現場作業における入力と確認のあり方を再設計しようとする試みである。
現状はプロトタイプであるが画面表示、入力、保存、クラウド連携、音声フィードバックまでを一つの流れとしてつなげられている点には手応えがある。今後は実際の現場に近い運用を意識しながら開発を進めていく予定である。
